# 운영체제의 분류
> 학습목표
- 운영체제의 분류, 구조

## 운영체제가 하는 일
- 자원 관리 
    - 프로세스(스케줄링), 메모리, I/O, 파일
- 시스템 관리
    -  H/W, 네트워킹, 명령해석기, 시스템 오류/복구

- 운영제체의 지표
    - 편리성, 효율성(처리량, 반환시간, 신뢰도, 사용가능도), 제어 서비스

- 임베디드 동작과정(리눅스)
    - ROM 코드 로드 : SRAM으로 부트코드 로드
    - SPL 부팅 : 시스템 초기화, 프로그램을 주기억장치에 로드
    - TPL 부팅 : 전체 부트 로더 실행, 명령어 실행 등 유지관리
    - 커널 : 시스템에 상주 후 제어
    

# 운영체제 발전사
> 학습내용
- 년도별 O/S 주요 특징

## 운영체제 역사
- 최초의 컴퓨터는 OS가 없어서 응용 프로그램이 하드웨어 규격을 준수
    - 주변기기 드라이버 필요
    - 복잡도가 높아져서 운영체제 필요성 대두

- 연도별 특징
    - 1940 
        - OS없음
        - 기계어로 처리, 단순 순차처리
            - 천공 카드 판독중 낭비되는 자원활용방안 필요
    
    - 1950
        - OS : 배치처리
        - 작업별 일괄처리 : 한번에 한프로그램 실행
        - 버퍼링 : 프로세스가 다음 작업을 미리 저장
        - 스플링 : 속도가 빠른 디스크를 버퍼로 사용 미리 입출력 장치 읽기
    
    - 1960
        - OS : 다중프로그래밍, 시분할 시스템, 다중 처리시스템, 실시간 처리시스템 구현
        - 가상 기억장치, 고급언어로 운영체제 개발, 데이터 통신 지원
    
    - 1970 초
        - OS : 다중모드 시스템, 범용 시스템
            - GUI 운영체제, 사용자에게 많은 기능 제공
        - 일괄처리, 시분할 처리, 다중 프로그래밍, TCP/IP표준 기술, 네트워크 보안
    
    - 1970 중
        - OS : 분산 처리 시스템
        - 응용프로그램 개발, 데이터 베이스활용, 네트워크 기술 발전, 펌웨어 적용
    
    - 1990 
        - OS : 병렬 및 분산 처리
        - 웹 등장, 분산 컴퓨터, GUI, 개인용 서버 운영체제 보편화
        
    - 2000 ~
        - OS : 모바일, 임베디드, 가상화, 클라우드 컴퓨팅(IaaS, PaaS, SaaS)
        - 네트워크 기반 분산/병렬처리 
    
- 운영기법
    - 1 세대
        - 일괄처리 : 일정량의 데이터를 한번에 처리
    
    - 2 세대
        - 다중 프로그래밍 : 하나의 CPU로 메모리에 여러 프로그램을 로드해서 처리
        - 시분할 처리 : CPU사용 시간을 분할(프로세스 스케줄링)
        - 다중처리 : 여러 CPU로 여러 작업 수행
        - 실시간 처리 : 작업을 약속된 시간내에 처리
    
    - 3 세대
        - 다중 모드 
    
    - 4 세대
        - 분산 처리 : 네트워크를 이용 여러 대의 컴퓨터를 사용해서 작업처리
