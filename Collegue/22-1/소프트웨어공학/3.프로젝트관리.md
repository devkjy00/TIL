## SI방식의 소프트웨어 개발 프로젝트
- 프로젝트 관련 액터
    - 발주자 -> 계약서 -> 수주자 -> 용역수행

- 수행 절차
    1. 입안 
        - 발주자 : 프로젝트 구상, 개요서 작성, 발의, 내부승인
    
    2. 발주 
        - 발주자 : RFP 작성 및 배포
        - RFI(Request For Information) : 기술 흐름이나 제품 정보를 취합하기 위해 관련 업체에게 보내는 요청서, 생산/서비스 업체에 제품에대한 설명을 요청하는 것
        - RFP(Request For Proposal) : 고객이 요구사항을 문서로 작성(프로젝트 목적, 범위, 일정, 예산..)
    
    3. 제안 
        - 발주자 : 제안 내용에 대한 문의 응답
        - 수주자 : 제안서 작성
        - 제안서
            - 목적, 기업소개, 내용(솔루션, 일정, 예산, 품질보증)
            - 프로젝트 수행방법(개발/관리 방법론, 테스트 전략, 조직 구성, 인력)
    
    4. 평가 
        - 발주자 : 제안서 평가
        - 수주자 : 제안 발표
        - 대부분 경쟁입찰, 가격/기술 평가로 종합점수 산출해서 우선협상대상자 선정
    
    5. 협상 
        - 우선협상 대상자 선정 및 조정
        - 제안가격 조정(출혈경쟁), 범위와 솔루션 재조정, 일정협의, 품질기준 조정 등
    
    6. 계약 
        - 계약서에 날인 및 프로젝트 시작
        - 프로젝트 수행 계획서, 수주업체 관련 증빙 서류, 투입인력 보안각서 등
    

## 프로젝트의 성공과 실패
- 프로젝트 성공 기준
    - 범위(기능구현)
        - 기술적/안전/성능 요구사항
        - 비즈니스/프로젝트 관리/인도 요구사항
    
    - 일정(납기)
    - 예산
        - 예산 내역(재료비, 인력 인건비, 부대비용등)

- 임계경로

## 프로젝트 관리
- 정의 : 지식, 스킬, 도구 및 기법을 프로젝트 활동에 적용하는 것
- 프로젝트 관리 지식체계(PMBOK)
    - 통합관리 : 계획, 조정, 총괄 관리
        - 프로젝트 관리 계획서 작성

    - 범위관리 : 범위 설정
        - 사업수행계획서, 요구사항정의서, 요구사항추적매트릭스, WBS 작성
    
    - 일정관리 : 단위 작업 분할, 작업별 일정 관리
        - 일정관리계획서 작성

    - 비용관리 : 작업 단위별로 할당, 부대비용, 재료비등 관리
        - 견적지침서, 규모견적서, 비용산적내역서, 비용/예산관리계획서 작성
    
    - 품질관리 : 산출물의 요구사항 충족기준 설정, 관리
        - 품질보증계획서, 프로젝트기준서, 관리/개발산출물표준양식, 형상관리계획서, 검토결과서 작성
    
    - 인력관리 : 프로젝트 구성원의 활동을 조직, 조정
        - 인력투입계획서 작성
    
    - 의사소통관리 : 이해당사자(Stakeholder)간에 정보 전달체계를 관리
        - 착수보고서, 단계말검토보고서, 완료보고서, 주간업무보고서 작성
    
    - 위험관리 : 위험요인 발견, 분석, 대책 수립
        - 리스크관리대장, 리스크평가/대처계획서 작성
    
    - 조달관리 : 필요한 자원 확보, 관리
        - 제안요청서, 입찰안내서 작성
    
- 프로젝트 관리 프로세스
    1. 착수단계(Initiating) 
    2. 기획단계(Planning)
    3. 실행단계(Executing)
    4. 통제단계(Monitoring and Controlling)
    5. 종료단계(Closing)

## 성과관리기법
- 용어정의
    - EVM(Earned Value Method)
        - 프로젝트 일정, 범위, 비용등의 계획과 실적을 화폐단위로 환산, 평가하는 기법

    - BCWS(Budgeted Cost of Work Scheduled, 계획비용)
        - 일정상의 작업을 완료하는데 필요한 예산, PV(Planned Value)
    
    - BCWP(Budget Cost of Work Performed, 획득비용)
        - 수행된 작업에 대하여 할당된 예산, EV(Earned Value, 획득가치)
    
    - ACWP(Actual Cost of Work Performed, 실제비용)
        - 수행된 작업에 대해 지출된 실제금액, AC(Actual, Value)
    
- 공식
    - SV(Schedule Variance, 일정차이)
        - EV-PV, 0이면 준수
    - CV(Cost Variance, 원가차이)
        - EV–AC, 0이면 준수
    - SPI(Schedule Performance Index) 
        - EV/PV, 1이면 준수
    - CPI(Cost Performance Index)
        - EV/AC, 1이면 준수

## 품질 비용
- 준수비용
    - 예방 비용(Prevention Cost)
        - 예방을 위한 결함원인과 행동을 정의하는데 지출되는 비용
    - 평가 비용(Appraisal Cost)
        - 규격을 만족하는가를 확인하기 위한 품질 측정, 평가에 드는 비용
    
- 비준수비용
    - 내부실패비용(Internal Failure Cost)
        - 고객에게 전달되기 전, 규격에 맞지 않아 수정/실패 진단하는 비용
    
    - 외부실패비용(External Failure Cost)
        - 고객에게 전달된 후, 제품이나 서비스를 수정하는데 드는 비용

## 일정계획
- WBS(Work Break-down Structure, 작업분할구조도)
    - 프로젝트 업무내역을 가시화, 책임과 역할이 분명
    - 필요 인력과 일정 계획, 개발비 산정의 기초로 활용

- PERT/CPM(Program Evaluation Review Technique/Critical Path Method)
    - CPM(활동목록)작성 -> CPM 네트워크 작성
        - WBS의 작업 순서, 기간등을 네트워크(방향그래프)로 표현
        - 중점관리를 해야하는 작업이 명확해진다
    
    - ES(Earlist Strat Time) : 가능한 빨리 시작할 수 있는 시간
    - EF(Earlist Finish Time) : ES에 시작했을때 가장빠른 완료시간
    - LS(Latest Start Time) : 일정지연을 막기위한 가장 늦은 시작시간
    - LF(Latest Finish Time) : LS에 시작했을때 허용가능한 작업소요시간
    - ST(Slack Time) : 느슨한, 여유있는 시간

- **임계경로**
    - CPM 네트워크에서 ES,EF,LS,LF,ST등을 고려해 가장 빨리 작업을 완료할 수 있는 경로

- 간트차트(Gantt chart)
    - 프로젝트 일정관리를 위한 바(bar) 형태의 도구
