# 객체 지향언어
- 객체지향이론의 기본 개념은 '실제 세계는 사물로 이루어져있으며, 발생하는 모든 사건들은 사물간의 상호작용이다.'라는 것
- 코드 간에 서로 관계를 맺어 줌으로써 보다 유기적으로 프로그램을 구성하는 것이 가능해졌다
- 너무 객체 지향개념에 얽매여서 고민하기 보다는 프로그램을 기능적으로 완성한 다음 객체 지향적으로 코드를 개선하는 것이 좋다

# 클래스와 객체
- 객체의 구성요소
    - 객체가 가지고 있는 속성과 기능을 그 객체의 멤버라고 한다

- 인스턴스의 생성
    - 선언 : 클래스명 변수명;
    - 생성 : 변수명 = new 클래스명();
    - 인스턴스가 메모리의 빈공간을 생성후 멤버변수는 각 자료형에 기본값으로 초기화한다
    - 인스턴스는 참조변수로만 다룰 수 있고, 참조변수 타입과 인스턴스 타입이 일치해야한다

- 객체 배열
    - 생성 : 클래스명[] 변수명 = new 클래스명[길이];
    - 객체 배열도 같은 타입의 객체만 저장할 수 있다

# 변수와 메서드
- 변수 종류
    - 클래스 변수
        - static이 붙은 것은 클래스 변수
        - 클래스가 메모리에 올라갈 때 생성된다 
            - 생성된후 프로그램이 종료될때 까지 유지된다
        - 클래스이름.클래스변수 로 접근가능하다

    - 인스턴스 변수
        - 일반 멤버 변수
        - 인스턴스가 생성될때 생성된다

    - 지역 변수
        - 멤버 변수를 제외한 나머지 변수
        - 변수 선언문이 수행되었을때 생성된다
        - 블럭{}을 벗어나면 소멸된다(for, while도 포함)

- 메서드
    - 높은 재사용성, 중복된 코드 제거, 프로그램의 구조화
    - 처음에 프로그램을 설계할 때 내용이 없는 메서드를 작업단위로 만들고 하나씩 완성해가는 것도 좋은 구조화 방법이다
    - 선언부
        - 변수의 타입을 생략할 수 없다
        - 메서드 이름은 동사로 만든다
        - 반환값이 없는 경우 'void'
            - void인 경우 컴파일러가 메서드 마지막에 'return;'을 자동 추가한다
    - 매개변수의 타입
        - call by value : 기본형 매개변수는 값이 복사된다
        - call by reference : 참조형,배열 매개변수는 주소값이 복사된다
            - 참조형변수와 배열변수는 주소값을 가진다
            - 참조형 매개변수를 활용하면 반환 값 없이도 메서드의 실행결과를 얻어 올 수 있다

    - 구현부
        - return문은 반환타입과 일치하거나 자동형변환이 가능해야한다
            - 단하나의 값만 반환 할 수 있다
    - 참조형 반환타입
        - 모든 참조형 타입의 값은 '객체의 주소' 이므로 정수값이 반환 된다

    - 메서드 호출
        - 인자의 타입은 매개변수의 타입과 일치하거나 자동 형변환이 가능해야한다
        - static메서드는 같은 클래스 내의 인스턴스 메서드를 호출 할 수 없다

- JVM의 메모리 구조
    - 3가지 주요영역: method area, call stack, heap
    - 메서드 영역
        - 클래스에 대한 정보를 저장, 클래스변수가 생성되는 영역
    - 힙 영역
        - 인스턴스에 대한 정보를 저장, 인스턴스 변수가 생성되는 영역
    
    - 호출 스택
        - 메서드의 작업에 필요한 메모리 공간을 제공한다
        - 메서드가 호출되면 메모리를 할당, 수행하는 동안 지역변수(매개변수)들과 연산의 중간결과등을 저장 작업을 마치면 메모리공간을 반환





- 재귀호출(recursive call)
    - call by value 는 메서드와 호출한 메서드와 관계없이 독립적인 작업 수행이 가능하다
        - 매개변수의 증감을 정의해서 내부호출하면 된다
    - 재귀 호출도 조건문이 필수이다
    - 매개변수 복사와 종료 후 복귀할 주소저장 등의 작업으로 반복문보다 재귀 호출의 수행시간이 더 오래 걸린다
        - 그럼에도 사용하는 이유는 논리적 간결함 때문이다

- 클래스 메서드, 인스턴스 메서드
    - 인스턴스 메서드 -> 인스턴스 변수를 필요로 하는 메서드
    - 클래스 메서드 -> 인스턴스와 관계없는 메서드
    - 멤버변수는 인스턴스 변수와 static 변수를 모두 통칭하는 말이다
    - 클래스 메서드 
        1. 클래스를 멤버 변수 중 모든 인스턴스에 공통으로 사용하는 것에 static을 사용
            - 모든 인스턴스에서 같은 값이 유지되어야 하는 변수는 static을 붙여서 클래스 변수로 정의해야 한다
        
        2. 클래스 메서드는 인스턴스 변수를 사용할 수 없다.
        3. 메서드내에서 인스턴스 변수를 사용하지 않는 다면, static을 붙이는게 조핟

- 클래스 멤버와 인스턴스 멤버간의 참조와 호출
    - 클래스 멤버가 인스턴스 멤버를 참조, 호출할 때는 블럭안에 인스턴스를 생성해야한다
        - 클래스 멤버가 존재하는 시점에 인스턴스 멤버가 존재하지 않을 수 있기 때문에
    
- int result = new 클래스().메서드();
    - 생성하고 바로 메서드를 사용할 수 있다

# 오버로딩(overloading)
- 오버로딩이란
    - 매개변수의 개수 또는 타입이 다르면, 같은 이름을 메서드를 정의 할 수 있다
    - 한 클래스 내에 같은 이름의 메서드를 여러개 정의 하는 것을 메서드 오버로딩이라고한다

- 조건
    - 메서드 이름이 같아야 한다
    - ***매개변수의 개수 또는 타입이 달라야 한다***
        - 타입이 다른 매개변수의 선언 순서가 다른 경우 오버로딩으로 간주하지만 지양해야한다

- 예 : println 메서드는 인자의 타입에 따라서 다른 메서드를 호출한다

- 가변인자
    - 선언: '타입... 변수명' (int... x)
    - 메서드를 호출 할 때마다 내부적으로 배열을 새로 생성한다
        - 꼭 필요한 경우에만 사용하는 것이 좋다
        - 배열을 매개변수로 선언하면 반드신 인자를 지정해줘야 하지만 가변인자는 값을 지정하지 않아도 괜찬다
    - 제일 마지막에 선언 해야 한다
    - 가변인자를 사용한 메서드는 오버로딩 하지 않는 것이 좋다
        - 구분하기 어렵다


# 생성자(Constructor)
- 인스턴스 초기화 메서드로 클래스내에 선언되고 리턴값이 없다
    1. 생성자의 이름은 클래스의 이름과 같아야한다
    2. 생성자는 리턴 값이 없다

- 생성자도 오버로딩이 가능하다
- 연산자 new가 인스턴스를 생성하고 생성자는 변수를 초기화하는 역할을 한다
- 기본 생성자(default constructor)
    - 생성자가 정의되어 있지 않으면 컴파일러가 자동으로 기본 생성자를 추가한다
        - 클래스이름() {빈 블럭}
    
    - 클래스의 접근 제어자가 public인 경우에는 기본 생성자도 public이 된다
    
- 매개변수가 있는 생성자
    - 클래스가 작성할 때 다양한 매개변수를 가진 생성자를 오버로딩하고 제공해서 인스턴스를 바로 초기화 하는 것이 바람직하다

- this(), this
    - 생성자의 이름으로 클래스이름 대신 this()를 사용한다
        - 한 생성자에서 다른 생성자를 호출 할 때는 반드시 첫줄에서만 호출 가능하다
        - 초기화가 된 상태에서 다른 생성자를 호출하면 이전 초기화가 무의미해진다
        - 생성자들이 서로 호출하도록 하여 유기적으로 연결해주면 더 좋은 코드를 얻을수 있다
    
    - 매개변수의 이름과 인스턴스변수의 이름이 같으면 this로 구분하면된다
        - this.x = x
        - x = x 는 둘다 지역변수로 간주한게 된다
        - this는 참조변수로 인스턴스 자신을 가리킨다
        - 모든 인스턴스 변수에는 this가 지역변수로 숨겨진 채로 존재한다

- 인스턴스를 생성할 때 고려사항
    1. 어떤클래스의 인스턴스를 생성할 것인가
    2. 어떤 생성자로 인스턴스를 생성할 것인가

# 변수의 초기화
- 선언과 동시에 적절한 값으로 초기화 하는 것이 바람직하다
    - 멤버변수는 초기화를 하지 않아도 자동적으로 기본값으로 초기화가 된다
        - 배열이 값들을 초기화하는 것과 같다
    - 지역변수는 사용하기 전에 반드시 초기화해야한다

- 멤버변수의 초기화 방법
    - 명시적초기화 
        - 변수를 선언과 동시에 초기화하는것
        - 가장 우선적으로 고려되어야 한다
    
    - 초기화 블럭
        - 클래스 초기화 블럭 : static{}, 인스턴스 초기화 블럭 : {}
        - 초기화 작업이 복잡한 경우 사용하는게 좋다
        - 클래스 초기화 블럭은 메모리 로딩때 한번, 인스턴스 초기화는 생성될 때마다 수행된다
            - 클래스가 메모리에 로딩될 때 클래스 초기화 블럭이 먼저 수행되고 그다음 main수행된다

    - 초기화 순서
        - 클래스 변수 : 기본값 -> 명시적초기화 -> 클래서 초기화 블럭
        - 인스턴스 변수 : 기본값 -> 명시적초기화 -> 인스턴스 초기화 블럭 -> 생성자
